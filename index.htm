All core features from the HMS requirements document.

Fully aligned with FR-1 to FR-11 plus additional AI capabilities.

FR-1

Hospital Self-Registration

Hospitals self-onboard by sharing name, address, contact & license details. Tenant ID is auto-generated and admin account created instantly.

FR-2

Multi-Tenant Isolation

Each hospital has isolated data (schema-per-tenant model). Tenant context comes from JWT token; cross-tenant access is blocked via middleware.

FR-3

Secure OAuth2 + JWT

Access and refresh tokens with tenant ID, user ID, roles and permissions ensure secure, scalable authentication.

FR-4/5

RBAC + ABAC

Role-Based (SUPER_ADMIN, HOSPITAL_ADMIN, DOCTOR, NURSE, etc.) and Attribute-Based access (department, shift, confidentiality) for fine-grained control.

FR-6/7

User & Password Management

Admin-managed users with strong password policy, reset flows, and secure session invalidation on change.

FR-8/9

Patient Lifecycle

OPD/IPD patient registration, unique patient IDs, search by ID, name, phone, plus department and doctor filters.

FR-10

Prescription Management

Doctors record medicines, dosage, frequency, duration, and reuse templates for common prescriptions.

FR-11

Dynamic Role-Based UI

Menus and screens adapt automatically to each role‚Äôs permissions ‚Äî both on frontend and backend level.
